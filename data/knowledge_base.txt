Machine learning is commonly divided into three main types: supervised learning, unsupervised learning, and reinforcement learning. Supervised learning uses labeled examples. Unsupervised learning finds patterns in unlabeled data. Reinforcement learning trains agents through rewards and penalties.

Overfitting occurs when a model learns the training data too well, including noise and accidental details. Such a model performs well on training data but generalizes poorly to new data. Regularization helps prevent overfitting by adding L1 or L2 penalty terms to the loss function, discouraging overly complex models. Cross-validation splits data into multiple folds so model performance can be estimated more reliably. The bias-variance tradeoff describes the balance between underfitting from high bias and overfitting from high variance; the goal is to minimize total error.

Backpropagation computes gradients of the loss with respect to each neural network weight by applying the chain rule of calculus. Common activation functions include ReLU, sigmoid, and tanh. Convolutional Neural Networks, or CNNs, are specialized for processing grid-like data such as images. Pooling layers in CNNs reduce spatial dimensions, lowering computation while retaining important features. LSTM networks address the vanishing gradient problem with gating mechanisms that control information flow.

The transformer architecture was introduced in "Attention Is All You Need" in 2017. Transformers use self-attention mechanisms and process entire sequences in parallel. Self-attention allows a model to weigh the importance of different words relative to each other in a sequence. Word embeddings represent words as dense vectors where semantically similar words are geometrically close. Transfer learning in NLP typically means pre-training large models on massive corpora and then fine-tuning them on downstream tasks. BERT uses bidirectional transformer training with masked language modeling and next sentence prediction.

GPT is a family of autoregressive language models trained to predict the next token given previous tokens, using massive text datasets. Instruction tuning fine-tunes pre-trained LLMs on instruction-following datasets to better align with human intent. RLHF, or Reinforcement Learning from Human Feedback, uses human preferences to align LLMs to be helpful, harmless, and honest. Chain-of-thought prompting encourages LLMs to show reasoning step by step and can improve performance on complex tasks. GPT-4 supports up to 128K tokens of context in some versions.

Retrieval-Augmented Generation, or RAG, combines generative LLMs with retrieval from external knowledge bases to produce grounded, up-to-date answers. A RAG pipeline usually has a retriever that searches a document store and a generator, usually an LLM, that produces answers from the query and retrieved passages. Dense retrieval uses neural embeddings to encode queries and documents, determining relevance by cosine similarity. Chunking strategy matters because it affects retrieval precision and context window fit; common options include fixed-size, semantic, and hierarchical chunking. Advanced RAG techniques include re-ranking, query expansion, HyDE, and iterative retrieval for multi-hop questions.

Vector databases store and query high-dimensional vector embeddings for fast similarity search. FAISS is a library for efficient similarity search that supports exact and approximate nearest neighbor algorithms. Text embeddings convert text into numerical vectors where semantically similar texts produce geometrically close vectors. HNSW builds a hierarchical graph for logarithmic-complexity approximate nearest neighbor search. Hybrid search combines dense vector search with sparse keyword search such as BM25, often using Reciprocal Rank Fusion.

LangChain is an open-source framework for building LLM applications with abstractions for chains, agents, and data connections. LangChain Expression Language, or LCEL, is a declarative way to compose chains using the pipe operator, supporting streaming, async execution, and batching. LangGraph extends LangChain for stateful multi-actor applications as directed graphs, including workflows with cycles. LangChain memory types include ConversationBufferMemory, ConversationSummaryMemory, ConversationWindowMemory, and vector store memory. LangChain retrievers fetch relevant documents from a data source given a query and support vector stores and BM25.

LangSmith is a platform for debugging, testing, evaluating, and monitoring LLM applications through automatic tracing. LangSmith traces capture inputs, outputs, latency, token usage, and errors for every component in a chain. LangSmith Prompt Hub is a repository for storing, versioning, and sharing prompt templates; it supports A/B testing and rollback. LangSmith helps monitor production LLM applications with latency percentiles, error rates, token costs, and feedback annotations. LangSmith datasets are used for systematic evaluation with example inputs and expected outputs to compare model versions or prompt changes.

RAGAS is an open-source framework for evaluating RAG pipelines using LLM-based reference-free and reference-aware metrics. RAGAS computes faithfulness by extracting claims from an answer and checking whether each claim can be inferred from the retrieved context. Answer relevancy measures how well the answer addresses the original question by generating synthetic questions from the answer. Context recall measures how well the retrieved context covers the information needed, evaluated against a ground truth reference. RAGAS evaluation requires user queries, generated answers, retrieved contexts as a list of passages, and optionally reference answers.

Guardrails AI is an open-source framework for adding validation and safety checks to LLM outputs with configurable on-fail actions. PII means Personally Identifiable Information; exposing it can violate privacy regulations such as GDPR and HIPAA. Structured output validation ensures LLM responses conform to expected schemas such as JSON, detecting and fixing invalid formats. Constitutional AI is a technique where a model is given principles to follow and uses self-critique to improve its responses. Common AI safety concerns with LLMs include hallucination, toxicity, bias, PII leakage, and jailbreaking attacks.
